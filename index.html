    As for inputting the content into WordPress using ACF to reuse the structure dynamically, the process would be something like:

    1. Create a new Field Group for the layout. Make the Field Type Flexible Content if using the pro version, or Group if using the free version. Flexible Content makes the layout have more modular flexibility, although Group should also work.
    2. Create sub-fields for the content in each column - title text, button text and button link for Column 1.
    3. Create a layout for the reusable Column 2 section that can be saved as its own block to be reused seperately. Create a new Field Group for this column and populate with a Repeater Field.
    4. Inside the Repeater Field, add an image field, text or number field for the number, and text field for the title, for each icon group.
    5. This should allow for Column 1 to be flexible and editible when used, and Column 2 to be dynamically reusable and customisable when needed to be used seperately.

  3. When should you use !important in your (S)CSS?

    Ideally, never. It overrides styles and could cause issues or make stylesheets harder to maintain. It is better to target specificity on elements where a change is needed.

  4. Which breakpoints would you use to test for responsiveness? Give reasoning for this.

    I test everything from 4k downwards. Most sites will usually have a max-width of 1920px or 1440px. A good start for breakpoints are 1200px at the largest sizes. For medium sized screens a breakpoint of 768px is ideal (and is also a good time to switch over to mobile navigation) and 480px for the smallest of screens. All content should be readable down to 320px width without the page overflowing.

    These are usually just guides, as it depends on the content on the page as to where and when it needs to break. Pages with a lot of dynamic, responsive content using modern css may need to break less. There may also be situations where a lot of breakpoints are required - and this will vary on a case by case basis.

  5. How would you make this section responsive / layout on mobile?

    How this would look on mobile would depend on the overall design of the site and how that section was implemented. The easiest way to adjust it for mobile would be to use flex and either flex-wrap: wrap or flex-direction: column. To save some horizontal space, the progress titles could be hidden and only display the icon to represent the status of the progress bar. If it's just an image it would be easy to swap it out by creating a class that only shows at a certain breakpoint. For example:

    @media screen and (max-width: 768px) {
  .large-img {
     display: none;    
  }  
  .small-img {
     display: block;    
  }
}
